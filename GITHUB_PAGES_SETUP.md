## Quick Setup (3 steps)

### Step 1: Initialize Git Repository (if needed)

```bash
# If not already a git repository
git init
git add .
git commit -m "Initial commit with documentation"
```

### Step 2: Create GitHub Repository

1. Go to https://github.com/new
2. Repository name: `MediaIngredientMech`
3. Description: "Curated Media Ingredient Ontology Mappings"
4. Public repository
5. Do NOT initialize with README (we already have files)
6. Click "Create repository"

### Step 3: Push and Enable GitHub Pages

```bash
# Add remote (replace with your GitHub username/org)
git remote add origin https://github.com/YOUR_USERNAME/MediaIngredientMech.git

# Push to GitHub
git branch -M main
git push -u origin main

# Enable GitHub Pages (via GitHub web interface):
# 1. Go to repository Settings
# 2. Navigate to "Pages" section
# 3. Source: Deploy from a branch
# 4. Branch: main
# 5. Folder: /docs
# 6. Click "Save"
```

**Your site will be published at:**
```
https://YOUR_USERNAME.github.io/MediaIngredientMech/
```

## Alternative: GitHub CLI

If you have `gh` CLI installed:

```bash
# Create repository
gh repo create MediaIngredientMech --public --source=. --remote=origin

# Push code
git push -u origin main

# Enable Pages via web interface (gh CLI doesn't support Pages API yet)
# Or use GitHub API:
gh api repos/{owner}/{repo}/pages -X POST -f source='{"branch":"main","path":"/docs"}'
```

## Customization

### Update Site Metadata

Edit `docs/_config.yml`:

```yaml
title: Your Project Name
description: Your description
theme: jekyll-theme-minimal  # or another theme
```

### Update GitHub Links

Edit `docs/index.html` and change:

```html
<a href="https://github.com/YOUR_USERNAME/MediaIngredientMech">View on GitHub</a>
```

### Regenerate After Data Changes

After updating ingredient data:

```bash
# 1. Export collection to individual files
just export-individual

# 2. Rebuild documentation
just build-docs

# 3. Commit and push
git add docs/
git commit -m "Update documentation with latest data"
git push
```

GitHub Pages will automatically rebuild and deploy (usually within 1-2 minutes).

## Continuous Deployment Workflow

### GitHub Actions (Optional)

Create `.github/workflows/deploy-docs.yml`:

```yaml
name: Deploy Documentation

on:
  push:
    branches: [ main ]
    paths:
      - 'data/**'
      - 'src/mediaingredientmech/schema/**'
      - 'scripts/**'

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3

      - name: Set up Python
        uses: actions/setup-python@v4
        with:
          python-version: '3.11'

      - name: Install dependencies
        run: |
          pip install -e ".[dev]"

      - name: Build documentation
        run: |
          just build-docs

      - name: Commit updated docs
        run: |
          git config --local user.email "github-actions[bot]@users.noreply.github.com"
          git config --local user.name "github-actions[bot]"
          git add docs/
          git diff-index --quiet HEAD || git commit -m "Auto-update documentation"
          git push
```

This automatically rebuilds documentation when data or schema changes.

## Troubleshooting

### Issue: 404 on GitHub Pages

**Solution:** Ensure:
1. Repository is public (or you have GitHub Pro for private repos)
2. Pages is enabled in Settings → Pages
3. Source is set to `main` branch, `/docs` folder
4. Files are committed and pushed to `main` branch

### Issue: JSON not loading in browser

**Solution:**
1. Check `docs/data/ingredients.json` exists
2. Verify file size is reasonable (<1MB is good)
3. Check browser console for CORS errors
4. Use a web server for local testing (not `file://`)

### Issue: Markdown not rendering

**Solution:**
1. Ensure `_config.yml` has correct markdown settings
2. Check `.nojekyll` file exists if using custom HTML
3. Regenerate with `just gen-docs`

### Issue: Schema docs out of date

**Solution:**
```bash
# Regenerate schema documentation
gen-doc --directory docs src/mediaingredientmech/schema/mediaingredientmech.yaml

# Rebuild browser data
python scripts/browser_export.py

# Commit and push
git add docs/
git commit -m "Regenerate documentation"
git push
```
